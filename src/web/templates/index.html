How to read this benchmark

Distributed engines like PySpark have structural scheduling costs, including JVM startup, partition management, executor coordination, and local file-commit requirements. This saved local benchmark was generated on Windows, where Spark can initialize with Java but reliable local Parquet writes require Hadoop native binaries. The benchmark therefore records Pandas timings and marks Spark as unavailable for local auto mode. In a production review, this is the honest conclusion: choose Pandas for small local data, and choose Spark when distributed scale, fault tolerance, and partitioned processing justify the overhead.
